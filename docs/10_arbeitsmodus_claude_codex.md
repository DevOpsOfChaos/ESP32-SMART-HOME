# Arbeitsmodus für Claude und Codex

## Ziel
Externe Assistenten sollen auf derselben Grundlage arbeiten und nicht jedes Mal heimlich neue Regeln einführen.

## Feste Regeln
- Architektur nicht neu erfinden
- Altprojekt nicht als Basis nutzen
- Public/Private-Trennung einhalten
- jede größere Änderung dokumentieren
- keine stille Sonderarchitektur pro Gerät
- deutsche Kommentare im Code
- klare Abschnittsstruktur

## Pflicht bei jeder sinnvollen Änderung
- `CHANGELOG.md` ergänzen
- neue Datei in `PROTOKOLL/` anlegen
- offene Punkte sichtbar dokumentieren

## Prioritäten
1. saubere Architektur
2. keine privaten Daten
3. gute Dokumentation
4. testbare Struktur
5. erst danach Feature-Ausbau
