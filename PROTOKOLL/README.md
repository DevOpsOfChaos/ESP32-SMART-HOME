# PROTOKOLL

Hier wird jede relevante Änderung als eigener Stand dokumentiert.

## Pflichtformat
Jeder neue Stand bekommt eine neue Datei:
- `beta01_...txt`
- `beta02_...txt`
- usw.

## Inhalt je Datei
- Anlass
- Änderung
- Begründung
- Verbesserungseffekt
- Risiken / Trade-offs
- Teststatus
- Lehrerblick: Eigenleistung vs. Übernahme
